ETAP 0: GitHub i Przygotowanie Środowiska
Repozytorium musi istnieć min. 2 tygodnie.

Zadanie: Utwórz repozytorium CertControlSystem.

Prompt dla AI (do wygenerowania plików startowych):

"Tworzę projekt w ASP.NET Core MVC (.NET 8).

Przygotuj mi zawartość pliku .gitignore typową dla projektów Visual Studio i .NET.

Napisz treść pliku README.md, który opisuje projekt 'System Zarządzania Certyfikatami'. Opis ma zawierać informację, że projekt realizuje wzorzec MVC, wykorzystuje SQL Server, Entity Framework oraz procesy w tle do wysyłki powiadomień Email/SMS."

ETAP 1: Baza Danych (SQL Server)
Fundament systemu. Tworzymy schemat spełniający wymogi 4 encji i relacji.

Narzędzie: SQL Server Management Studio (SSMS).

Zadanie: Uruchom poniższy skrypt (możesz poprosić AI o jego modyfikację, ale ten jest gotowy do użycia).

Prompt dla AI (lub gotowy skrypt do wklejenia):

"Przygotuj skrypt T-SQL tworzący bazę danych CertDB. Wymagania:

Clients: Id (PK), FirstName, LastName, Email, PhoneNumber, CreatedAt.

CertificateTypes: Id (PK), Name (np. 'BHP', 'SEP'), DefaultValidityMonths.

Certificates: Id (PK), ClientId (FK), TypeId (FK), IssueDate, ExpirationDate, IsActive.

NotificationLogs: Id (PK), CertificateId (FK), Channel (Email/SMS), SentDate, MessageContent, Status.

Dodaj przykładowe dane (INSERT) do każdej tabeli (min. 3 klientów, 3 typy certyfikatów, 5 certyfikatów), aby od razu mieć na czym pracować."

ETAP 2: Inicjalizacja Projektu i Scaffold (Reverse Engineering)
Tworzymy aplikację i pobieramy strukturę z bazy do kodu C#.

Zadanie w Visual Studio: Utwórz nowy projekt -> ASP.NET Core Web App (Model-View-Controller).

Prompt dla AI (Instrukcja konfiguracji):

"Mam istniejącą bazę danych SQL Server o nazwie CertDB. Stworzyłem pusty projekt ASP.NET Core MVC w .NET 8.

Jakie paczki NuGet muszę zainstalować, aby użyć Entity Framework Core z podejściem Database-First (Scaffolding)? (Podaj komendy Install-Package).

Podaj mi dokładną komendę Scaffold-DbContext, która wygeneruje pliki klas (Entities) oraz Context w folderze Models. Mój Connection String to: Server=(localdb)\MSSQLLocalDB;Database=CertDB;Trusted_Connection=True;."

ETAP 3: Logika MVC – Kontrolery i Widoki (CRUD)
To jest główna część zaliczenia "Projektu MVC". Musisz stworzyć formularze i listy.

Zadanie: Wygenerowanie kontrolerów z widokami. Visual Studio ma do tego automat (Prawy klilk na Controllers -> Add -> Controller -> MVC Controller with views, using Entity Framework).

Jeśli chcesz kod od AI – Prompt:

"Mam wygenerowany CertDbContext i modele Client, Certificate. Proszę, napisz kod dla ClientsController:

Akcja Index wyświetlająca listę klientów w tabeli HTML.

Akcja Create (GET i POST) z walidacją ModelState.IsValid.

Widok Create.cshtml ma używać Tag Helpers (np. asp-for) i klas Bootstrapa.

Akcja Details, która pokazuje dane klienta ORAZ listę jego certyfikatów (użyj .Include(c => c.Certificates)).

Następnie napisz CertificatesController:

Akcja Create, która pobiera listy rozwijane (DropDownList) dla ClientId i CertificateTypeId (przekaż je przez ViewBag lub ViewModel)."

ETAP 4: Autoryzacja i Użytkownicy (ASP.NET Identity)
Zamiast JWT (które jest do API), użyjemy wbudowanego mechanizmu Identity, który ma gotowe widoki logowania/rejestracji.

Zadanie: Dodanie obsługi logowania.

Prompt dla AI:

"Chcę dodać autoryzację do mojego projektu MVC.

Jak dodać ASP.NET Core Identity do istniejącego projektu (korzystając z gotowego UI biblioteki)?

Jak zmodyfikować mój Program.cs, aby obsłużyć logowanie?

Chcę, aby tylko zalogowany użytkownik mógł dodawać i edytować dane ([Authorize]), ale listę certyfikatów (Index) mógł widzieć każdy. Pokaż jak udekorować kontrolery atrybutami.

(Opcjonalnie) Jak stworzyć domyślnego użytkownika 'Admin' przy starcie aplikacji (Seed Data)?"

ETAP 5: API REST (Wymóg "Sieciowy" + "API CRUD")
Mimo że mamy MVC, specyfikacja wymaga "API CRUD". Zrobimy jeden kontroler API w tym samym projekcie.

Zadanie: Stworzenie endpointu, który zwraca czysty JSON (np. dla zewnętrznych systemów lub Twojego serwisu powiadomień).

Prompt dla AI:

"Potrzebuję spełnić wymóg posiadania REST API w projekcie MVC. Stwórz osobny kontroler w folderze Controllers/Api o nazwie CertificatesApiController.

Ma dziedziczyć po ControllerBase i mieć atrybut [ApiController].

Metoda GET /api/certificates ma zwracać listę certyfikatów w formacie JSON.

Stwórz proste DTO (CertificateDto), aby nie zwracać zapętlonych danych z bazy.

To API posłuży mi później do integracji z systemem powiadomień."

ETAP 6: Serwis Powiadomień (Background Service + SMTP + API SMS)
To jest element na ocenę 5.0 z przedmiotu sieciowego oraz kluczowa funkcjonalność biznesowa.

Architektura: Stworzymy HostedService wewnątrz aplikacji ASP.NET, który działa w tle niezależnie od klikania w stronę.

Prompt dla AI:

"Chcę dodać proces w tle (IHostedService / BackgroundService), który sprawdza ważność certyfikatów.

Stwórz klasę NotificationWorker.

Logika: Raz na 24h sprawdź w bazie certyfikaty, które wygasają za 30 dni.

Dla każdego znalezionego: a) Wyślij Email (napisz prostą metodę używając MailKit i SMTP Gmaila). b) Zapisz informację w tabeli NotificationLogs.

Pokaż jak zarejestrować ten serwis w Program.cs.

(Dla ambitnych) Jak wywołać zewnętrzne API (np. SMSAPI) używając HttpClient w tym serwisie?"
